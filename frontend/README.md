# Telegram Mini App frontend

Vite + React + TypeScript shell для Telegram Mini App.

## Команды

```bash
npm install
npm run dev
npm run build
```

Локальный dev server: `http://localhost:5173/miniapp/`.

API в dev режиме проксируется на `http://127.0.0.1:8000`.

## Ограничения этапа 6

1. Реализован shell, auth bootstrap, Telegram WebApp adapter, темы и нижняя навигация.
2. Секреты во frontend не добавляются.

## Этап 7

Реализованы пользовательские экраны:

1. Согласие одной кнопкой.
2. Форма записи.
3. Выбор типа встречи и длительности.
4. Выбор даты и слота.
5. Review перед отправкой.
6. Список заявок.
7. Карточка заявки.
8. Отмена.
9. Подготовка переноса.
10. Визуальные уведомления.

Статический preview доступен по `http://localhost:5173/miniapp/`, если запущен
`preview-server.mjs`. Он не заменяет Vite, а нужен только для быстрой визуальной проверки.

## Этап 8

Реализованы админские экраны:

1. Сводка по заявкам.
2. Список заявок с фильтром статуса.
3. Карточка заявки.
4. Подтверждение pending-заявки со ссылкой на встречу.
5. Отклонение pending-заявки с причиной.
6. План подтвержденных встреч.
7. Просмотр настроек расписания.
8. Просмотр рабочих часов.
9. Добавление и удаление закрытого дня.
10. Добавление типа встречи и переключение активности.

Ограничения этапа 8:

1. Админская отмена подтвержденной встречи не добавлена, потому что отдельное
   backend-правило `cancel_by_admin` пока не реализовано.
2. Редактирование рабочих часов и базовых настроек расписания пока read-only:
   backend на этом этапе отдает только `GET` для этих данных.
3. Блокировка пользователей и отправка сообщения пользователю не входят в MVP
   frontend без отдельных backend endpoints.
4. `npm run build` нужно проверить после исправления локального `npm install`.

Статический preview этапа 8 доступен по `http://localhost:5173/miniapp/`, если
запущен `preview-server.mjs`. Он не заменяет Vite, а нужен только для быстрой
визуальной проверки.

## Production

Dockerfile собирает Mini App через Node/Vite stage и копирует результат в
`frontend/dist` внутри Python-образа. FastAPI отдает build по пути
`MINI_APP_PUBLIC_PATH`, по умолчанию `/miniapp`.

Production URL: `https://calendar.finforbiz.pro/miniapp/`.

В `.env.production` указывается только путь:

```bash
MINI_APP_FRONTEND_DIST_PATH=frontend/dist
```

Секреты во frontend не добавляются.
